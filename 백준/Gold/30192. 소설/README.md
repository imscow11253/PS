 ### 풀이 

 <p>정말 어려웠다. 우선 브루트포스는 절대 아니라고 생각했지만 브루트 포스였다. 단, 좀 변형을 해주어야 한다. k 길이 이상의 연속하는 문자열의 시작-끝 index를 vector에 다 저장해두고 L을 1부터 N 까지 반복하면서 해당 연속하는 문자열이 해결이 되는지 안되는지 판단했다. 근데 이 판단하는 로직이 꽤나 복잡하게 나왔다. 

판단하는 로직이 총 4가지 분기문으로 나왔는데 각각 설명해보면

1. 연속하는 문자열이 주어진 문자 가장 처음인 경우로, 이때는 연속하는 문자열 길이가 k 보다 크고 (사실 이건 체크 안해도 되긴한다. vector에 check하고 넣기 때문), i가 k보다 크면 안된다. 
2. sta%i ≠ 0 이라는 뜻은 i로 문자열을 나누었을 때 연속하는 문자열 중에서 앞부분 집합에 짤려서 들어가는 부분이 있다는 것이다. |abc|ccc| 이런 경우를 말한다. 여기서는 연속하는 c가 4개인데 맨 처음 c가 abc 집합으로 짤린다. 즉 이런 경우가 있을 때, 앞 집합에 들어가는 연속된 문자의 개수가 k보다 크면 안된다는 것이다. i-(sta%i) 가 앞 부분 집합에 들어가는 연속하는 문자열의 개수다. 
3. 똑같이 앞 부분이 짤리지만 이번엔 뒷부분을 check하는 로직이다. (end - sta+1)-(i-(sta%i)) 이 식은 뒷 부분에 해당하는 문자열의 개수를 말한다. 이게 k보다 크면서, i가 k보다 크면 안된다.
4. 마지막은 앞부분이 짤리지 않는다는 것이다. 여기서는 뒷부분만 보면 되고, 뒷부분이 k보다 크면서 (이건 vector에 넣을 때 check  하기 때문에 안해도 된다.) i가 k보다 크면 안된다. 

분기 처리를 하는 부분이 좀 어려웠지만 차근차근하게 풀었다. 풀이를 찾아보려고 했지만 없어서 그냥 내가 풀지 뭐 라는 마인드 풀었다. 뿌듯하다.</p>
